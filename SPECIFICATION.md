# imechen 仕様書

## 概要

画像/動画形式を完全オフラインで変換するためのツール。

---

## 機能仕様

各保存先設定はメモリに保存され、次回起動時にも反映される

### 共通UI仕様

#### ファイルリスト表示
- ファイル選択後、サムネイル + ファイル名 + メタ情報（解像度・サイズ）をリスト表示
- 各ファイルに削除（×）ボタン付き

#### 保存先選択
- テキストボックス + フォルダを開くアイコンボタン
- デフォルト保存先は設定画面から変更可能

#### 出力結果表示
- 変換完了後、出力ファイル名・サイズ削減率・出力パスを表示
- 「出力先を開く」ボタンでOSのファイルマネージャーを開く

### 静止画像変換機能

変換元画像と保存先を選んで変換する

それぞれを相互に変換できる機能を提供する
- jpeg
- png
- gif
- webp
- bmp
- HEIF (変換元)

#### (optional) リサイズ

変換前にサイズを指定することで、リサイズすることが出来る

- 幅×高さの自由入力
- 「縦横比維持」トグル付き（デフォルト: ON）
  - ON: 一方の値を変更すると、もう一方が自動計算される
  - OFF: 幅・高さを個別に自由入力可能（縦横比が変わる）

#### (optional) 圧縮とクオリティ設定

png の場合は oxipng + pngquant

他の画像形式の場合も、クオリティ指定でサイズ圧縮できるなら適切なツールを使って圧縮する

### 動画変換機能

変換元動画と保存先を選んで変換する

それぞれを相互に変換できる機能を提供する
- mp4
- mov
- avi
- webm
- flv
- HEVC (変換元)

画質・解像度変更も可能
(画質を上方に変更することは不可)

#### 解像度設定

- プリセット選択（1080p, 720p, 480p 等）またはカスタム自由入力（幅×高さ）
- 動画読み込み時、元動画の解像度がデフォルト値として自動入力される
- 元の解像度を超える値は設定不可（上方変更の禁止）

### 連番画像->アニメーション作成機能

連番画像を読み込んで、アニメーションを作成する機能
(保存先を選択できる)

出力時に、必要であればfpsも指定する

出力形式は以下の通り
- GIF
- awebp
- mp4

### 設定変更

#### テーマ

- ダーク / ライト / システム連動の3択
- OS設定連動（システムテーマに自動追従）

#### デフォルト保存先

- 全変換機能で共通のデフォルト保存先を設定可能

## 技術スタック

| レイヤー | 技術 |
|---|---|
| フレームワーク | Tauri |
| フロントエンド | Bun + Solid.js (TypeScript) + Vite |
| バックエンド | Rust |
| メディア処理 | `ffmpeg` crate |
| 永続化 | JSONファイル（`settings.json`, `history.json`） |
| 動画コーデック | H.264 |

## デザイン方針

**Mix: Glassmorphism + Neumorphism 2.0** のハイブリッドスタイルを採用。

### スタイルルール

| UI要素 | スタイル | 実装 |
|---|---|---|
| カード / リストアイテム | Glassmorphism | 半透明背景(`rgba(255,255,255,0.05)`) + `backdrop-filter: blur(16px)` + 1px半透明ボーダー |
| 入力エリア | Glassmorphism | 半透明背景 + ブラー + 半透明ボーダー |
| 操作ボタン（pause/cancel等） | Neumorphism | 凹凸シャドウ（ライト/ダーク2方向） |
| プライマリボタン（DL開始等） | Gradient + Glow | `linear-gradient` + `box-shadow`グロウ |
| プログレスバー | Gradient | `linear-gradient(#F97316, #FB923C)` |
| セレクト / ドロップダウン | Neumorphism | 凹凸シャドウ + ボーダー |
| サイドバー | Solid | ソリッド背景（`--bg-inset`） |
